Current workflow

Evidence before automation.

LingoMux reuses a trustworthy embedded text track before creating new timing. Every stage can run alone, and the working path can run as one recorded job.

01

Inspect

Read streams, chapters, codecs, language tags, and track flags.

02

Select

Prefer a known source-language text subtitle already inside the media.

03

Translate

Send scene-sized cue groups with nearby dialogue and a glossary.

04

Check text

Compare cue IDs, timing, numbers, glossary choices, and language rules.

05

Mux

Add a titled Matroska subtitle track without encoding video or audio.

06

Verify

Reopen the file, round-trip the subtitle, hash packets, and decode media.

CLI examples

Plan it. Run it. Prove it.

The same core supports a one-command job and separate stage commands. Start with a plan when you need to inspect the selected source and media behavior.

Read-only decision

See the selected source before changing a file.

The plan shows each stage, flags provider access, and reports if a missing stage prevents the run.

  • No file writes
  • No provider calls
  • No media encoding
uv run lingomux plan film.mkv \
  --from en \
  --to fa

Recorded pipeline

Create the selectable track and its audit record.

The command writes a new MKV, an editable SRT sidecar, and a versioned JSON manifest.

  • Scene context and glossary
  • Temporary sibling outputs
  • Verification before final move
uv run lingomux run film.mkv \
  --from en --to fa \
  --title Persian \
  --model MODEL_NAME \
  --glossary glossary.json \
  --allow-network \
  --output film.fa.mkv

Independent proof

Check a finished container again.

Verification can compare the translated sidecar, packet hashes for original media, and a decoded sample.

  • Stream and chapter inventory
  • Track language and title
  • Subtitle round trip
uv run lingomux verify \
  film.mkv film.fa.mkv \
  --language fa \
  --title Persian \
  --subtitle film.fa.srt \
  --hash-media

Translation quality

Context helps. Checks keep it honest.

A fluent sentence can still lose a name, number, cue, or timing boundary. LingoMux separates contextual translation from deterministic source-to-target checks.

A / translate

Dialogue is sent in scene-sized groups.

The provider receives stable cue IDs, nearby dialogue, the requested register, and preferred glossary terms. It never controls timestamps or paths.

cue IDsdialogueregisterglossary
B / compare

Clear factual changes block the output.

  • Missing, unexpected, or reordered cue IDs
  • Changed cue timing or numeric values
  • Missing target values from an explicit glossary
  • Unchanged source text and repeated output warnings

Persian profile

Right-to-left text stays in logical Unicode order.

The language checks cover Persian Yeh and Kaf, ZWNJ placement, bidi controls, NFC text, line length, and reading speed.

00:14:08,220 → 00:14:10,900

کاپیتان، ۱۲٫۵ ثانیه صبر کن.

Captain, wait 12.5 seconds.

Automation has a boundary. A fluent speaker still needs to review publication-ready Persian while watching the scene.

Media verification

A new track should not disturb the old ones.

The mux stage maps every original stream and uses stream copy. The verification stage then compares what went in with what came out.

01

Inventory

Original stream count, order, codec, language, title, and flags.

02

Subtitle

New track metadata plus extracted text and timing against the SRT.

03

Packets

Optional SHA-256 packet hashes for each original video and audio stream.

04

Decode

FFmpeg decodes a requested sample of the finished audio and video.

film.fa.lingomux.jsonschema 2
{
  "status": "complete",

  "source_stream_index": 2,
  "target_language": "fa",
  "translation_error_count": 0,
  "lint_error_count": 0
}

The manifest stores hashes and provider metadata, not subtitle text, glossary contents, request bodies, or credentials.

Roadmap

Build outward from a tested media path.

The project starts with embedded text because it provides the strongest timing evidence. Audio generation and subtitle acquisition come next, followed by review tooling and more formats.

  1. Now

    Embedded text pipeline

    Inspection, extraction, contextual translation, integrity checks, Persian lint, Matroska muxing, verification, and a job manifest.

  2. Next

    Audio and source acquisition

    Local speech recognition, word alignment, opt-in provider search, stronger synchronization, and target cue fitting.

  3. Later

    Review and delivery

    Waveform review, locked edits, OCR, WebVTT, ASS, MP4 packaging, and library integrations.

Questions

Before you run it.

Does LingoMux overwrite the source video?

No. The source path cannot be the output path. New media is written to a temporary sibling, verified, then moved to the requested destination.

Does it burn text into the picture?

No. The current muxer adds a selectable Matroska subtitle track and copies the existing video and audio streams.

Can it create subtitles when none exist?

Not yet. Local speech recognition is the next major source path. The current one-command run requires an embedded text subtitle.

Does an automatic check guarantee a good translation?

No. It can catch structural and factual problems, but dialogue choices, humor, register, and cultural adaptation still need a fluent reviewer.
