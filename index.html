Workflow

Use the strongest evidence first.

Existing subtitle timing is usually more valuable than another transcription pass. LingoMux chooses a source based on what the media already contains, then records every transformation in a resumable job manifest.

  1. Inspect

    List audio, subtitle, chapter, and attachment streams before making a plan.

  2. Select

    Reuse embedded text, check a permitted provider, or transcribe the selected audio.

  3. Synchronize

    Measure offset, drift, speech overlap, and release mismatches.

  4. Translate

    Translate scene-sized context with stable cue IDs and a user glossary.

  5. Validate

    Check timing, reading speed, language rules, names, and untranslated fragments.

  6. Mux and verify

    Add a selectable track, retain original streams, and inspect the finished container.

Design principles

Media changes should be explicit.

The tool is designed around inspectable plans and typed pipeline stages. Translation providers and speech models can change without controlling paths, timestamps, or media commands.

01 / selectable

Soft subtitles by default

Matroska is the first output target. Burned-in subtitles remain an explicit export option.

02 / local

Network access stays opt-in

Local processing is preferred. A plan shows what would leave the computer before any remote request.

03 / contextual

Translation sees the scene

Character names, register, nearby dialogue, and glossary terms inform translation before cue fitting.

04 / verified

The output is checked

Stream inventory, subtitle metadata, cue timing, and decode errors are checked after muxing.

Roadmap

Start with a trustworthy media path.

Persian is the first target language because it exposes right-to-left rendering, Unicode normalization, and translation-quality problems that generic subtitle wrappers often ignore.

First release

Inspect media, extract text subtitles, validate Persian cues, add a selectable MKV track, and verify stream preservation.

Next

Add contextual translation providers, local speech recognition, global offset correction, and drift detection.

Later

Add provider search, piecewise alignment, image-subtitle OCR, and a desktop review application.

The project is under active development.

The command interface shown here is the planned first release. Follow the repository for implementation progress and early test builds.
